# 08. 코딩 에이전트용 메인 프롬프트

아래 요구사항을 기준으로 새 프로젝트 폴더를 생성하고, 그 폴더 안에서 작업을 이어가 주세요.

## 프로젝트 개요
- 프로젝트명: 전도 파트너 플랫폼
- 목적: 지역교회와 연계한 전도 모임 운영용 웹/앱 플랫폼
- 대상 테넌트:
  - 안국역 전도 파트너
  - 삼성역 전도 파트너
  - 판교역 전도 파트너
  - 다산역 전도 파트너

## 기술 스택
- Flutter
- Firebase Authentication
- Cloud Firestore
- Firebase Storage
- Cloud Functions for Firebase
- Firebase Hosting

## 중요 제약사항
- 저는 코딩을 못하는 사용자입니다.
- 설명은 쉬운 한국어로 해 주세요.
- React는 사용하지 마세요.
- Flutter + Firebase만 사용해 주세요.
- 하나의 엔진(백엔드)에서 4개 테넌트를 운영하는 구조로 설계해 주세요.
- tenantId 기반 멀티테넌트 구조로 구현해 주세요.
- 앱/웹 모두 지원해야 합니다.
- MVP부터 개발해 주세요.
- 한 번에 전체 기능을 만들지 말고 단계별로 진행해 주세요.

## 1차 MVP 기능
1. 회원가입 / 로그인
2. 사용자별 tenantId 처리
3. 홈 화면
4. 공지 목록 / 상세
5. 모임 목록 / 상세 / 신청
6. 내 신청 내역
7. 관리자 최소 페이지
8. 동시 접속 제한 게이트 화면

## 동시 접속 제한 기능 요구
- 테넌트별 maxConcurrentUsers 설정 가능
- 초기값 100명
- 상한 초과 시 앱 종료 금지
- 게이트 화면을 보여 주세요
- 문구:
  - 현재 동시 접속 허용 인원은 100명입니다.
  - 현재 접속자가 많아 잠시 입장이 제한됩니다.
  - 아래 버튼을 눌러 현재 접속 현황을 다시 확인해 주세요.
  - 입장 가능 시 자동으로 다음 화면으로 이동합니다.
- 버튼명: 현재 접속자 재확인
- 재확인 버튼 클릭 시 최신 상태 재조회
- 입장 가능 상태가 되면 자동으로 메인 화면 이동
- 단순 조회가 아니라 동시 입장 충돌을 막을 수 있도록 원자적 처리 구조로 설계해 주세요.

## 권한 구조
- user: 일반 사용자
- operator: 지역 운영자
- admin: 전체 관리자

## 원하는 산출물
1. 프로젝트 폴더 구조
2. Flutter 초기 프로젝트 코드
3. Firebase 연동 구조
4. Firestore 컬렉션 구조 제안
5. 보안 규칙 초안
6. 실행 방법
7. 현재 단계에서 필요한 Firebase 콘솔 작업 목록
8. 다음 단계 작업 계획

## 코드 작성 규칙
- 폴더 구조를 먼저 제안한 뒤 코드 작성
- 너무 복잡한 패턴 사용 금지
- 빌드 가능한 상태 유지
- 더미 데이터 → Firebase 연결 순으로 진행 가능
- 생성/수정 파일 목록을 항상 정리해 주세요.
