# 07. 코딩 에이전트 작업 규칙

## 공통 원칙
1. Flutter + Firebase만 사용한다.
2. React 또는 별도 웹 프레임워크를 추가하지 않는다.
3. MVP 기준으로 구현한다.
4. 과도한 추상화 금지.
5. 복잡한 아키텍처 패턴 남용 금지.
6. 한 번에 모든 기능을 만들지 말고 단계적으로 진행한다.
7. 더미 데이터로 먼저 흐름을 검증하고 Firebase 연동을 붙인다.

## 코드 원칙
- 명확한 파일명 사용
- 기능 단위 폴더 분리
- 재사용 가능한 위젯 분리
- Firebase 접근 로직은 service/repository 계층으로 분리
- 하드코딩 최소화

## 보안 원칙
- Firestore Security Rules 반드시 작성
- tenantId 기반 권한 분리
- admin/operator/user 역할 분리
- 클라이언트에서 role 신뢰 금지, 서버/규칙에서 검증

## 품질 원칙
- 빌드 가능 상태 유지
- 화면 전환 흐름 명확화
- 에러 메시지 사용자 친화적으로 작성
- null 처리/예외 처리 필수

## 산출물 원칙
각 단계 완료 시 아래를 반드시 제출하게 요청:
1. 생성/수정 파일 목록
2. 전체 폴더 구조
3. 실행 방법
4. 구현 범위 요약
5. 다음 단계 제안
