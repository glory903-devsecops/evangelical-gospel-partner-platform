# 05. 동시 접속 제한 게이트 설계

## 목표
과도한 동시 접속으로 인한 장애를 줄이기 위해 테넌트별 동시 접속 상한을 설정하고, 상한 초과 시 본 서비스 화면 진입을 차단한다.

## 기본 정책
- 기본 허용 인원: 100명
- 추후 설정값 변경으로 1000명까지 확장 가능
- 앱 종료 금지
- 게이트 화면 유지
- 재확인 버튼 제공
- 허용 상태가 되면 자동 입장

## 사용자 문구 예시
현재 동시 접속 허용 인원은 100명입니다.
현재 접속자가 많아 잠시 입장이 제한됩니다.
아래 버튼을 눌러 현재 접속 현황을 다시 확인해 주세요.
입장 가능 시 자동으로 다음 화면으로 이동합니다.

버튼: 현재 접속자 재확인

## 동작 흐름
1. 로그인 후 tenantId 확인
2. access gate check 함수 호출
3. currentActiveUsers < maxConcurrentUsers 이면 입장 승인
4. 아니면 게이트 화면 표시
5. 사용자가 재확인 버튼 클릭 또는 자동 재조회
6. 허용 가능 시 세션 슬롯 확보 후 메인 화면 진입

## 기술 주의사항
- 단순 조회 후 입장 금지
- 반드시 원자적 처리 필요
- 트랜잭션 또는 Cloud Function에서 슬롯 선점 처리
- heartbeat 또는 세션 만료 설계 필요

## 추가 요구
- 자동 재조회 주기: 5초 또는 10초
- 재확인 버튼 수동 호출 허용
- 접속 가능 시 자동 이동
