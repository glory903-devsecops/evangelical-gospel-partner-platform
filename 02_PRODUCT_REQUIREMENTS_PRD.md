# 02. 제품 요구사항 문서 (PRD)

## 1. 제품 목표
Flutter + Firebase 기반으로 iPhone / Android / Web에서 동작하는 멀티테넌트 전도 파트너 플랫폼을 개발한다.

## 2. 사용자 유형
### 2.1 일반 사용자
- 회원가입
- 자기 지역 플랫폼 접근
- 공지 확인
- 모임 신청
- 내 신청 내역 확인

### 2.2 지역 운영자
- 지역 공지 작성/수정
- 지역 모임 생성/수정/마감
- 신청자 목록 확인
- 지역 사용자 관리(최소)

### 2.3 전체 관리자
- 전체 테넌트 관리
- 테넌트별 설정 관리
- 동시 접속 상한 설정
- 운영자 권한 부여

## 3. 기능 요구사항
### 3.1 인증
- 이메일 기반 회원가입/로그인
- 비밀번호 재설정
- 로그인 유지

### 3.2 테넌트 구조
- 사용자 계정은 1개 주 테넌트에 속함
- 테넌트별 데이터 접근 분리
- 안국역/삼성역/판교역/다산역 각각 독립 공간처럼 보여야 함

### 3.3 공지
- 테넌트별 공지 목록
- 공지 상세 보기
- 운영자/관리자만 공지 등록 가능

### 3.4 모임 신청
- 모임 목록 조회
- 모임 상세 보기
- 모임 신청 / 신청 취소
- 신청 마감 / 정원 체크

### 3.5 관리자 기능
- 테넌트별 설정값 변경
- 운영자 지정
- 동시 접속 제한 인원 설정

### 3.6 접속 제한 게이트
- 현재 동시 접속 허용 인원 초과 시 서비스 본 화면 접근 차단
- 차단 화면에서 현재 접속자 재확인 버튼 제공
- 재확인 시 최신 상태 조회
- 허용 범위가 되면 자동 입장

## 4. 비기능 요구사항
- 초기 1인 운영 가능해야 함
- 구조가 단순해야 함
- 최소한의 비용으로 유지 가능해야 함
- 테넌트 데이터가 섞이면 안 됨
- 모바일과 웹에서 동일한 핵심 흐름 제공
- 앱 강제 종료 방식 금지, 게이트 화면 방식 사용

## 5. MVP 성공 기준
- 4개 테넌트 중 1개만 우선 오픈해도 전체 구조가 유지될 것
- 사용자 가입, 공지 조회, 모임 신청, 관리자 설정 변경이 정상 동작할 것
- 접속자 제한 기능이 동작할 것
